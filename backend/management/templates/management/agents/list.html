Agent Configurations

What are Agents?

Agents are the AI backends that process user queries in the chat interface. When a user sends a message, the active agent receives it, calls the appropriate MCP tools (database queries, documentation search, schema lookups), and composes a natural-language response.

TetherDust abstracts agent backends so you can swap providers without changing the rest of the system.

Agent Types

  • Codex — OpenAI Codex CLI integration. Django POSTs chat requests to the codex service container, which runs codex exec with MCP tool access. Best for environments already using the Codex CLI.
  • Claude Code — Anthropic Claude Code CLI integration. Django POSTs chat requests to the claude service container, which runs claude -p with MCP tool access. Authenticated by a Claude Pro/Max subscription OAuth token (from claude setup-token).

Authentication Methods

Choose the method that matches your agent type and infrastructure:

  • Token — Passes a bearer token directly to the agent CLI at invocation time.
  • API Key — Sets the provider's environment variable (e.g. ANTHROPIC_API_KEY) in the agent service before it spawns the CLI.
  • Device — Uses OAuth device-code flow to authenticate interactively. Codex-only. The login is cached on disk.
  • Guided sign-in — Claude Code: the management runs claude setup-token on the agent container, you approve in a browser and paste back the short authorization code. The resulting subscription token is stored encrypted.

System Prompt

The system prompt is injected into every conversation to shape agent behavior. A well-crafted prompt should include:

  • The agent's role and purpose (e.g. "You are a data assistant for Acme Corp.")
  • Instructions on when to use each MCP tool — especially to call get_query_examples before writing custom SQL
  • Output formatting expectations (tables, plain text, etc.)
  • Any domain-specific rules or terminology

The system prompt is the single biggest lever for improving response quality.

Active Agent

Only one agent can be active at a time. The active agent handles all incoming chat messages. Inactive agents are stored but unused — click the INACTIVE badge to activate one. Activating a new agent automatically deactivates the current one.

Tip: Create and test a new agent before activating it. Use the edit form to verify auth credentials and system prompt without affecting live users.
