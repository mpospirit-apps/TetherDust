General Settings

How Settings Work

These settings are stored in the database, not in environment variables or code. When you save a value here, it takes effect immediately — no server restart is required.

If a field has never been saved, TetherDust falls back to the corresponding environment variable from your docker-compose.yml. Once you save a value through this page, the database value takes precedence over the environment variable.

Service URLs

Codex Service URL is the address of the Codex API gateway that the web app uses to communicate with the AI agent. MCP Base URL is the address of the MCP server that exposes database tools to the agent. These default to the Docker service hostnames (http://codex:8002 and http://tdmcp:8001) and rarely need changing unless you are reconfiguring your deployment.

Timeouts

Doc Gen and Chart Gen timeouts control how long TetherDust waits for the AI to finish generating documentation or charts before giving up. The default is 1800 seconds (30 minutes). Increase this if generation tasks are being cut off prematurely on large datasets.

Agent Limits

Max Row Limit sets the global default for how many rows the agent may return per query when a user has no assigned role. Role-level limits always take precedence — this only applies when there is no role to inherit a limit from.

Hot Reload Interval controls how often the MCP server re-reads documentation sources from disk. Leave blank to disable hot reloading.

Tip: To revert a setting back to the environment variable default, you can clear the value and save — URL fields will fall back to the env var automatically. For integer fields, re-enter the default value manually.
