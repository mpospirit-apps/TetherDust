Adding an MCP Server

Remote HTTP

Register an MCP endpoint reachable over HTTP. Provide the full URL, transport type, and optional auth.

  • URL: e.g. https://mcp.example.com/mcp
  • Transport: streamable-http (current standard) or sse (legacy)
  • Bearer token: sent as Authorization: Bearer <token>. Encrypted at rest.
  • Extra headers: JSON object, e.g. {"X-API-Key": "abc"}

Local Subprocess

Run an MCP server as a child process inside TetherDust's local-mcp container. Ideal for packages like @notionhq/notion-mcp-server that require local auth tokens rather than OAuth.

  • Command: the executable, e.g. npx or uvx
  • Args: JSON array, e.g. ["-y", "@notionhq/notion-mcp-server"]
  • Env vars: JSON object of environment variables passed to the subprocess. Encrypted at rest. e.g. {"NOTION_API_KEY": "ntn_..."}

Who can use it

Custom MCP servers are opt-in per role. After saving, go to Roles → edit a role → check this server under "MCP Servers" to grant access.

Tip: Use the Test Connection button on the detail page to verify the server is reachable before assigning it to a role.
