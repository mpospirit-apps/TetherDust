Codebases

What are Codebases?

Codebases are source-code repositories that TetherDust exposes to AI agents as a source, alongside databases and documentation. The agent browses and reads files via the codebase MCP tools (list_codebases, get_codebase_tree, read_codebase_file, search_codebase) to answer questions about your code and to ground generated docs, dashboards, and tethers.

GitHub (on-demand)

The agent reads code live through the GitHub API — nothing is cloned to disk. On Sync, TetherDust validates access, resolves the default branch, and caches the repository's file tree so browsing is fast. File contents are fetched on demand.

Fields

  • Repository URL — e.g. https://github.com/owner/repo.
  • Branch — leave blank to use the repository's default branch.
  • Subpath — restrict to a sub-directory (useful for monorepos).
  • Include / Exclude globs — JSON arrays scoping which files the agent sees. Excludes default to common build output and binaries so the agent isn't flooded.
  • GitHub Token — optional, encrypted at rest. Required for private repositories; a read-only fine-grained PAT (contents: read) is sufficient. Even for public repos a token raises the API rate limit.
Note: The agent can read anything committed to the repository, including secrets that were committed by mistake. Only connect repositories whose full contents are safe for the agent (and your users) to see.
