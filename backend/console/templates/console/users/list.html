User Management

What are Users?

Users are everyone who logs in to TetherDust — both end users who interact through the chat interface and administrators who manage Mission Control. Each user account controls authentication, role assignment, and individual permission overrides.

Role Assignment

Every active user should have a role. A user without a role has no access to any tools or databases in the chat interface — they can log in, but queries will fail or return no results.

Roles define the permission boundary. Assign the most appropriate role first, then use per-user overrides for exceptions. See the Roles section for role configuration.

Staff Access

Staff users (Django is_staff) can access this Mission Control admin panel. Only grant staff status to administrators who genuinely need to manage system configuration.

Staff status is separate from role permissions — a staff user still needs a role for chat access. An admin who also uses the chat interface should have both staff status and an appropriate role.

Per-User Overrides

Beyond their role's default permissions, individual users can be granted access to specific additional tools or databases. This is useful for one-off exceptions without creating a dedicated role for a single person.

Overrides are additive — they extend role permissions but cannot remove them. To restrict a user below their role's level, assign them a more restrictive role instead.

Account Status

  • Active — The user can log in and use TetherDust normally.
  • Disabled — The user cannot log in. All their chat history, sessions, and audit log entries are preserved. Re-activate to restore access.

Creating a New User

Click + Add User to create a new account. You will set:

  • Username and initial password (the user can change this from their profile)
  • Email address (optional but recommended for notifications)
  • Role assignment
  • Staff status if the user needs admin panel access
Tip: Disable accounts rather than deleting them when a user leaves. Deletion removes their audit log entries and chat history, which may be needed for compliance or debugging.
