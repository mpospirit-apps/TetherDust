# First Steps

Once TetherDust is running and you can log in, this page walks you through the
essential configuration that turns a bare install into a working system: adding
a database, choosing an agent, connecting your documentation, and sending your
first question.

## The two surfaces

TetherDust has two web interfaces under the same login:

| Surface | URL | Who it's for |
|---|---|---|
| **Portal** | `/` | End users — chat, docs viewer, dashboards, reports, tethers |
| **Console** | `/management/` | Staff — configuration, generation, user management |

The superuser you created during installation is a staff user and has access to
both. Everything in this guide is done from the **management**.

## Step 1 — Add a database connection

Go to **Console → Databases → Add Connection**.

| Field | Guidance |
|---|---|
| **Name** | A short identifier the agent uses when calling `query_database`. Must be unique. |
| **Engine** | Choose from PostgreSQL, MySQL/MariaDB, ClickHouse, SQL Server, SQLite, Oracle, Snowflake, or BigQuery. |
| **Host / Port / Database** | Connection coordinates. Leave port blank to use the engine default. |
| **Username / Password** | **Use a read-only account.** See [[TetherDust Documentation/5. Security/2. Read-Only Enforcement.md|Read-Only Enforcement]] for setup scripts per engine. |
| **Read-only** | Checked by default. For PostgreSQL, MySQL, SQLite, Oracle, and ClickHouse this enables a session-level read-only mode in addition to SQL validation. |
| **Description** | Free text passed to the agent. Write something useful: what system owns this data, what it contains. |

After saving, click **Test Connection** to verify connectivity. The MCP server
picks up new connections immediately — no restart required.

## Step 2 — Configure an agent

Go to **Console → Agents → Add Agent**.

TetherDust supports multiple agent types; exactly one is **Active** at any time.
Switching is a database change — no restart needed.

| Agent type | Credential needed | Container |
|---|---|---|
| `codex` | ChatGPT Plus/Pro auth token | `codex` |
| `codex_api` | OpenAI API key | `codex-api` |
| `claude_code` | Claude Pro/Max OAuth token | `claude` |
| `claude_code_api` | Anthropic API key | `claude-api` |
| `direct_api` | Provider API key | None (in-process) |
| `ollama` | None (self-hosted) | `ollama` (profile) |

Fill in the credential, set **Active**, and save. The credential is encrypted at
rest using your `TETHERDUST_ENCRYPTION_KEY`. Only one configuration can be active
at a time — saving a new active agent deactivates the previous one.

For detailed setup per agent type, see
[[TetherDust Documentation/3. Agent Integrations/1. Overview.md|Agent Integrations]].

## Step 3 — Add documentation

The agent answers better when it has context about your database schema and
domain. Two things help:

### Documentation sources

Go to **Console → Docs → Add Source**. A documentation source points at a
folder under `documentations/` on disk. When you ran `docker compose up` for the
first time, TetherDust auto-discovered any folders already in that directory.

To add database documentation the agent can search:

1. Create a folder: `documentations/My Database/`
2. Add markdown files describing your tables, columns, and domain.
3. The source is auto-discovered on the next startup, or you can manually
   create a `DocumentationSource` record in the management pointing at the folder.

Set **Doc type** to `database` for schema/query documentation (enables
`get_query_examples` parsing) or `manual` for user guides and SOPs.

See [[TetherDust Documentation/2. Features/3. Docs.md|Docs]] for full details on
file formats, wiki-links, and Mermaid diagrams.

### System prompt

Go to **Console → Agents** and edit your active agent's **System Prompt**.
This is the global instruction sent to the agent with every request. A good
system prompt tells the agent about your data domain, preferred query style, and
any conventions it should follow.

See [[TetherDust Documentation/3. Agent Integrations/7. System Prompt.md|System Prompt]]
for guidance on writing an effective prompt.

## Step 4 — Create a role and a user

Unless every user should have unrestricted access, set up a
[[TetherDust Documentation/2. Features/11. Roles & Access Control.md|Role]] before
inviting users.

1. Go to **Console → Roles → New Role**.
2. Give it a name (e.g. `Analyst`).
3. Tick **Can chat**.
4. Select the **Allowed databases** this role may query.
5. Set a **Max row limit** (e.g. `500`).
6. Save.

Then go to **Console → Users → New User**, create the account, and assign the
role you just created.

---

## Step 5 — Ask your first question

Go to **Portal → Chat** (or log in as the new user you created and visit `/chat/`).

Try questions like:

- *"What tables do you have access to?"*
- *"How many records are in the [table] table?"*
- *"Show me the top 10 [entities] by [metric]"*

The agent calls `list_tables`, `get_table_schema`, and `query_database` behind
the scenes. You can see the tool calls it made in the assistant message after
the answer arrives.
