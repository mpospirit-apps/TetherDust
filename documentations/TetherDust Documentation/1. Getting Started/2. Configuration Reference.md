a `.env` file by Docker Compose. This page documents every variable, grouped by
concern.

## Required secrets

These two variables have no usable defaults — the stack will not function
correctly without them.

| Variable | Description |
|---|---|
| `TETHERDUST_ENCRYPTION_KEY` | Base64-encoded 32-byte Fernet key. Encrypts all stored database passwords and agent credentials. Generate: `python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"` |
| `DJANGO_SECRET_KEY` | Django's cryptographic signing key (sessions, CSRF, cookies). Generate: `python -c "import secrets; print(secrets.token_urlsafe(64))"` |

In `DEBUG=false` (production) mode, saving any credential without
`TETHERDUST_ENCRYPTION_KEY` raises `ImproperlyConfigured` and is refused. In
debug mode a warning is logged and the value is stored in plaintext.

## Django core

| Variable | Default | Description |
|---|---|---|
| `DJANGO_SECRET_KEY` | `django-insecure-change-me-in-production` | **Must be changed.** Used for session signing, CSRF, and cookie integrity. |
| `DJANGO_DEBUG` | `true` | `true` uses Django's dev server (auto-reload). `false` switches to the Daphne ASGI server and enables all production hardening. |
| `DJANGO_ALLOWED_HOSTS` | `localhost,127.0.0.1` | Comma-separated list of accepted `Host` headers. Add your domain for any non-local deployment. |
| `DJANGO_LOG_LEVEL` | `DEBUG` when debug, `INFO` otherwise | Python logging level for the app. |
| `DJANGO_SUPERUSER_USERNAME` | `admin` | Created on first boot if absent. |
| `DJANGO_SUPERUSER_PASSWORD` | `admin` | **Change before any real deployment.** |
| `DJANGO_SUPERUSER_EMAIL` | `admin@localhost` | Email for the bootstrapped superuser. |

## Django production hardening

These are only relevant when `DJANGO_DEBUG=false`. When debug is off,
`settings.py` automatically enables a set of security headers and transport
protections (assuming TLS is terminated in front of the app):

| Behaviour | Setting | Notes |
|---|---|---|
| HTTPS redirect | `SECURE_SSL_REDIRECT = True` | Redirects all HTTP to HTTPS. Set `DJANGO_SECURE_SSL_REDIRECT=false` if your proxy already handles this. |
| HSTS | `SECURE_HSTS_SECONDS = 31536000` (1 year) | Override with `DJANGO_SECURE_HSTS_SECONDS=0` while rolling out TLS. |
| Secure cookies | `SESSION_COOKIE_SECURE`, `CSRF_COOKIE_SECURE` | Both set to `True` automatically. |
| Content-type sniffing | `SECURE_CONTENT_TYPE_NOSNIFF = True` | Prevents MIME-type sniffing. |
| Proxy SSL header | `SECURE_PROXY_SSL_HEADER` | Set to `("HTTP_X_FORWARDED_PROTO", "https")`. Forward `X-Forwarded-Proto` from your reverse proxy. |

| Variable | Default (production) | Description |
|---|---|---|
| `DJANGO_SECURE_SSL_REDIRECT` | `True` | Set `False` if your proxy already redirects HTTP→HTTPS. |
| `DJANGO_SECURE_HSTS_SECONDS` | `31536000` | HSTS max-age in seconds. Set `0` to disable while validating TLS. |
| `DJANGO_CSRF_TRUSTED_ORIGINS` | *(empty)* | Comma-separated HTTPS origins for CSRF (required when behind a reverse proxy). Example: `https://tetherdust.example.com` |

Verify your production configuration with:

```bash
docker compose exec web python manage.py check --deploy
```

## Database (app DB)

These variables configure the PostgreSQL database that TetherDust itself uses
(not the target databases your users query). A single set of variables is shared
across `db`, `mcp`, `local-mcp`, `web`, `celery-worker`, and `celery-beat` via
Docker Compose environment substitution.

| Variable | Default | Description |
|---|---|---|
| `DB_HOST` | `db` | PostgreSQL host. In Compose this is the service name. |
| `DB_PORT` | `5432` | PostgreSQL port. |
| `DB_NAME` | `tetherdust` | Database name. |
| `DB_USER` | `tetherdust` | Database user. |
| `DB_PASSWORD` | `tetherdust` | Database password. **Change for any non-local deployment.** |

## Agent gateway URLs

These tell Django where to reach each agent gateway container. The defaults
match the in-stack service names and should not need changing unless you run
services on non-standard ports.

| Variable | Default | Description |
|---|---|---|
| `CODEX_SERVICE_URL` | `http://codex:8002` | URL for the Codex subscription-token gateway. |
| `CODEX_API_SERVICE_URL` | `http://codex-api:8002` | URL for the Codex API-key gateway. |
| `CLAUDE_SERVICE_URL` | `http://claude:8002` | URL for the Claude Code subscription-token gateway. |
| `CLAUDE_API_SERVICE_URL` | `http://claude-api:8002` | URL for the Claude Code API-key gateway. |

Each `AgentConfiguration` can override its gateway URL via `service_url`. The
env vars above are used only when `service_url` is left blank — they are
fallbacks, not the only routing mechanism.

## MCP server

| Variable | Default | Description |
|---|---|---|
| `MCP_BASE_URL` | `http://mcp:8001` | Base URL of the built-in MCP server. Used by agent gateways to register per-request tool-filter tokens. |
| `LOCAL_MCP_BASE_URL` | `http://local-mcp:8003` | Base URL of the local custom MCP server. |
| `LOCAL_MCP_PORT` | `8003` | Port the local MCP server listens on (inside its container). |
| `MCP_TRANSPORT` | `streamable-http` | MCP transport: `stdio`, `sse`, or `streamable-http`. |
| `MCP_HOST` | `0.0.0.0` | Bind address for the MCP server. |
| `MCP_PORT` | `8001` | Port for the MCP server. |
| `TETHERDUST_DOCUMENTATIONS_DIR` | `/app/documentations` | Filesystem path to the `documentations/` folder (inside the MCP container). |
| `TETHERDUST_MAX_ROW_LIMIT` | *(unlimited)* | Global cap on rows returned by `query_database`. Role-level `max_row_limit` is also applied; the lower of the two wins. |
| `TETHERDUST_HOT_RELOAD_INTERVAL` | — | Interval in seconds for hot-reloading documentation sources. |
| `TETHERDUST_LOG_LEVEL` | `INFO` | Log level for the MCP server. |

## Production checklist

Before exposing TetherDust to a network, work through:

- [ ] Set `TETHERDUST_ENCRYPTION_KEY` to a freshly generated value.
- [ ] Set `DJANGO_SECRET_KEY` to a freshly generated value.
- [ ] Change `DJANGO_SUPERUSER_PASSWORD` away from `admin`.
- [ ] Change `DB_PASSWORD` away from `tetherdust`.
- [ ] Set `DJANGO_DEBUG=false`.
- [ ] Set `DJANGO_ALLOWED_HOSTS` to your real hostname(s).
- [ ] Set `DJANGO_CSRF_TRUSTED_ORIGINS` to your HTTPS origin(s).
- [ ] Terminate TLS in front of the app (reverse proxy) and forward `X-Forwarded-Proto`.
- [ ] Run `docker compose exec web python manage.py check --deploy` — it should report no issues.
- [ ] Keep `.env` out of version control (it is already in `.gitignore`). Prefer a secrets manager for production environments.
