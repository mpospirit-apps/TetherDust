# Audit Log

TetherDust records three distinct audit trails — one for every database query
the agent executes, one for every AI documentation-generation run, and one for
every AI chart/dashboard-generation run. Staff can review all three from the
console. This page describes what is captured, where it is stored, and how to
read it.

## At a glance

| Log | What it records | Created by |
|---|---|---|
| `QueryAuditLog` | Every `query_database` MCP tool call — who queried, which database, success or failure. | `core/consumers/audit.py`, triggered by the chat WebSocket consumer. |
| `DocGenerationLog` | Every AI documentation-generation run — destination, doc type, status, agent output, errors. | `mcp_server/tools/create_documentation.py` via the MCP tool call. |
| `ChartGenerationLog` | Every AI chart/dashboard-generation run — dashboard name, charts created, status, agent output. | `core/engines/tether_engine.py` / dashboard generation flow. |

All three are retained indefinitely — there is no automatic pruning.

## Query Audit Log

**Model:** `core/models/connections.py` → `QueryAuditLog`

**Console path:** Console → Audit → Query Log

### Fields

| Field | Type | Description |
|---|---|---|
| `user` | FK → `auth.User` (nullable) | The user whose chat session triggered the query. `null` if the user was deleted. |
| `database` | FK → `DatabaseConnection` (nullable) | The target database connection. `null` if the connection was deleted. |
| `query` | Text | The SQL query text that was executed. Currently recorded as `[extracted from agent response]` — exact SQL extraction is captured via the MCP tool response. |
| `row_count` | Integer (nullable) | Number of rows returned. `null` if the query failed or count was not available. |
| `execution_time_ms` | Integer (nullable) | Query execution time in milliseconds. `null` if not recorded. |
| `success` | Boolean | `True` if the query returned results, `False` if it raised an error. |
| `error_message` | Text | Error details when `success=False`. Blank otherwise. |
| `created_at` | Timestamp | When the log entry was created. |

### What triggers an entry

An entry is created whenever the chat WebSocket consumer detects a query result
or query error in the agent's streamed response (`core/consumers/audit.py`):

- A turn containing `"Query Results"` → creates a successful entry.
- A turn containing `"Query execution error"` → creates a failed entry.

### Filtering

The console audit list supports filtering by outcome:

- **All** — default, most recent 200 entries.
- **Successes only** — `?success=1`
- **Failures only** — `?success=0`

## Doc Generation Log

**Model:** `core/models/agent.py` → `DocGenerationLog`

**Console path:** Console → Audit → Doc Generation Logs

### Fields

| Field | Type | Description |
|---|---|---|
| `user` | FK → `auth.User` (nullable) | Staff member who triggered the generation. |
| `agent` | FK → `AgentConfiguration` (nullable) | The active agent at the time of generation. |
| `destination` | String | Target folder path where the documentation was written (e.g. `My Database/`). |
| `filename` | String | Filename of the generated document (empty for library runs). |
| `doc_type` | String | Doc type (e.g. `database`, `manual`, `api`). Matches the `DocumentationSource.doc_type`. |
| `is_library` | Boolean | `True` when a full multi-file documentation library was generated (a folder tree), `False` for a single file. |
| `status` | Choice | `running` → `success` / `partial` / `failed` |
| `started_at` | Timestamp | When the generation run began. |
| `completed_at` | Timestamp (nullable) | When the run finished. `null` if still running. |
| `execution_time_ms` | Integer (nullable) | Total wall-clock time in milliseconds. |
| `source_databases` | JSON list | Database connections used as sources for this run. |
| `source_docs` | JSON list | Documentation sources used as context for this run. |
| `errors` | JSON list | Per-file or per-step error records. |
| `error_message` | Text | Top-level error message on failure. |
| `file_size` | Integer (nullable) | Size of the generated file in bytes. |
| `prompt_used` | Text | The full prompt sent to the agent for this run. |
| `agent_output` | Text | The complete raw output from the agent. |

### Status values

| Status | Meaning |
|---|---|
| `running` | Generation is in progress. |
| `success` | All files were generated without errors. |
| `partial` | Some files were generated; some failed. |
| `failed` | Generation failed before producing output. |

### Console detail view

Click any log entry to see the full prompt sent to the agent, the raw agent
output, per-file errors, and source databases/docs that were included as context.

**Model:** `core/models/dashboards.py` → `ChartGenerationLog`

**Console path:** Console → Audit → Chart Generation Logs

### Fields

| Field | Type | Description |
|---|---|---|
| `user` | FK → `auth.User` (nullable) | Staff member who triggered generation. |
| `agent` | FK → `AgentConfiguration` (nullable) | The active agent at generation time. |
| `dashboard_name` | String | Name of the dashboard being populated. |
| `status` | Choice | `running` → `success` / `partial` / `failed` |

| `completed_at` | Timestamp (nullable) | When generation finished. |
| `execution_time_ms` | Integer (nullable) | Total wall-clock time in milliseconds. |
| `source_databases` | JSON list | Database connections used as sources. |
| `source_docs` | JSON list | Documentation sources used as context. |
| `charts_created` | Integer (nullable) | Number of charts successfully created in this run. |
| `errors` | JSON list | Per-chart error records. |
| `error_message` | Text | Top-level error message on failure. |
| `prompt_used` | Text | The full prompt sent to the agent. |
| `agent_output` | Text | The complete raw agent output. |

## Viewing logs in the console

All three logs are under **Console → Audit** (staff only).

| Page | URL | Records shown |
|---|---|---|
| Query Log | `/console/audit/` | Last 200 query log entries, filterable by success |
| Doc Generation Logs | `/console/audit/docgen/` | Last 200 doc generation runs, filterable by status |
| Chart Generation Logs | `/console/audit/chartgen/` | Last 200 chart generation runs |

Each list shows the most important fields at a glance. Doc and chart generation
log entries link to a detail page with the full agent output and prompt — useful
for debugging failed or partial generation runs.

## Dashboard summary

The console **home page** (`/console/`) shows a live activity summary built from
`QueryAuditLog`:

- **Queries (24 h)** — total query count in the last 24 hours, with a delta vs. the previous 24-hour period.
- **Failed queries (24 h)** — failed query count, with delta.
- **Queries per database (7 d)** — a breakdown of query volume by `DatabaseConnection` over the past week.
- **Recent queries** — the 8 most recent log entries with user, database, and outcome.

This gives a quick health check of agent activity without having to open the
full audit log.
