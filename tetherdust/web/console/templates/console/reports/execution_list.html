What are Report Runs?

A report run is a single execution of a scheduled report. Every time a report fires — whether on its schedule or triggered manually — an execution record is created here with the outcome, timing, and row count.

Runs are read-only. They cannot be re-run from this page — go to the Reports section to trigger a manual run.

Status

  • SUCCESS — The report SQL executed without errors and returned results.
  • FAILED — The report encountered an error. Click View to see the full error trace and SQL that was attempted.
  • RUNNING — The report is currently in progress. Refresh the page after a few moments.

Columns

  • Report — The report definition that produced this run.
  • Started — Time elapsed since the run began.
  • Duration — Total time the SQL query took to execute in milliseconds.
  • Rows — Number of result rows returned by the query.
  • Triggered By — The user who manually triggered the run, or Scheduled for automatic runs.
Tip: If a report is consistently failing, open a run detail to see the exact error. Common causes are schema changes in the target database, credential expiry, or SQL that relied on a table that no longer exists.
