Reports

What are Reports?

Reports are pre-written SQL queries that execute automatically on a schedule against any connected database. They are designed for recurring data extracts, operational health checks, and distributing insights to users who should receive data without having to ask for it.

Reports run independently of the chat interface — no AI agent is involved in report execution.

Creating a Report

To set up a report:

  • Write a valid SELECT query targeting one of your connected databases.
  • Choose a schedule type and configure the timing.
  • Set access control to define which roles can view results.
  • Optionally configure a delivery method so results are pushed automatically.

Schedule Types

  • Manual — The report only runs when you click the play (▶) button. No automatic scheduling.
  • Daily — Runs once per day at a configured time (UTC).
  • Weekly — Runs once per week on a chosen day and time.
  • Monthly — Runs on a specific day of the month.
  • Interval — Runs repeatedly every N minutes (e.g. every 5 minutes, every hour).

Running a Report Manually

Click the ▶ (play) button next to any report to trigger an immediate run outside of its schedule. This is useful for testing a new report, refreshing data on demand, or re-running after a failed execution.

Execution History

Every run is recorded with its status (success / failed / running), start time, duration, row count returned, and any error details. View the full history in the Report Runs section of the Monitoring sidebar, or click into an individual report run for the complete output and error trace.

Active vs Inactive

Inactive reports are not scheduled to run automatically. Their configuration and history are preserved. Toggle the status badge to pause or resume a report without deleting it.

Tip: Use Manual schedule during development to validate your SQL before enabling automatic scheduling. Check Report Runs after each test to confirm the query returns expected data.
