Query Audit Log

What is the Audit Log?

The audit log is an immutable record of every SQL query executed through the query_database MCP tool. Every entry captures who ran the query, which database it targeted, the SQL text, whether it succeeded, how many rows were returned, and how long it took.

Logs are written automatically — there is nothing to configure. They cannot be edited or deleted through the UI.

Filtering

Use the All / Success / Failed filter buttons to narrow the view. Failed entries indicate queries that were rejected by the validation layer or that produced a database error. Review the query text for syntax issues or permission violations.

Columns

  • User — The TetherDust user whose chat session triggered the query.
  • Database — The database connection the query was executed against.
  • Status — OK for successful executions, FAIL for errors.
  • Rows — Number of rows returned, up to the user's role row limit.
  • Time (ms) — Round-trip query execution time in milliseconds.
  • Query — Truncated SQL text. Hover to see the full query.
Tip: A high volume of FAIL entries from a specific user often indicates a misconfigured role or an agent producing incorrect SQL. Check the user's role permissions and review their recent chat sessions.
