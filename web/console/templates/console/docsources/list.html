Documentation Sources

What are Documentation Sources?

Documentation sources are folders of markdown files that AI agents can search using the search_docs and get_table_schema MCP tools. They give agents the knowledge needed to answer questions accurately — without documentation, agents must guess meanings from raw column and table names.

Good documentation is the single most impactful thing you can do to improve query quality and reduce hallucinations.

Auto-Discovery

Every top-level folder inside documentations/ is automatically discovered as a documentation source. When you visit this page, TetherDust scans the directory and creates entries for any new folders. If a folder is removed, its source is automatically deactivated.

Role-Based Access

Assign documentation sources to roles to control which users can search them. Go to Roles and check the documentation sources each role should access.

How search_docs Works

When an agent calls search_docs, TetherDust performs a full-text search across all active documentation sources. The top matching excerpts are returned to the agent as context.

Validating a Source

Click Validate to confirm that the folder path exists, is readable by the server process, and contains at least one markdown file.

Active vs Inactive

Inactive sources are completely excluded from search_docs results. Use this to disable a source temporarily without losing its configuration.

Tip: Start with schema docs for your most-queried tables. Even a one-line description per column dramatically improves agent accuracy.
