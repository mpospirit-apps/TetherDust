Email (SMTP)

What is this for?

TetherDust can deliver report results by email. These settings configure the outgoing mail server that is used for that delivery. Without a valid SMTP configuration, scheduled reports with email delivery will fail silently.

Configuration Fields

  • SMTP Host — The hostname of your mail server (e.g. smtp.gmail.com, smtp.sendgrid.net).
  • SMTP Port — Typically 587 for TLS (recommended) or 465 for SSL. Port 25 is generally blocked by cloud providers.
  • Username / Password — Credentials for authenticating with the SMTP server. For Gmail, use an App Password rather than your account password.
  • Use TLS — Should be enabled when using port 587. Encrypts the connection after it is established (STARTTLS).
  • From Email Address — The address that appears in the From field of delivered emails. Must be authorised by your mail provider.
  • Max Rows in CSV Attachment — Caps the number of rows included in the CSV file attached to report emails. Useful for keeping attachments within mail size limits.

Testing Your Configuration

Use the Send Test Email button to verify that TetherDust can reach your SMTP server and send a message. The test email is sent to the email address on your admin account. Save your settings first, then test.

How Settings are Stored

SMTP credentials are saved in the database, not in environment variables. The password is encrypted at rest using Fernet symmetric encryption. Leaving the password field blank when saving preserves the existing stored password.

Tip: For Gmail, enable 2-factor authentication on your Google account, then generate an App Password under Security → App passwords. Use that 16-character password here instead of your regular account password.
