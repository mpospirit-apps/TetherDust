# 0.4.7 — 2026-06-26

**Deferred fast-follow #7 (Docs): Mermaid diagram rendering.** From the 0.4.0
API-only + SPA migration's deferred-items list.

## Upgrade notes for admins
- No DB migrations and no new env vars. Rebuild the `frontend` image
  (`docker compose up --build`) to pick up the changes (a new `mermaid`
  front-end dependency is bundled).

## Added
- **Mermaid diagrams render as SVG in the Docs viewer.** ` ```mermaid ` fenced
  blocks (flowcharts, sequence, ER, class diagrams, …) now render as actual
  diagrams instead of highlighted code. This materially improves the AI-generated
  database/codebase docs, whose prompts lean heavily on Mermaid. Diagrams follow
  the light/dark theme, and a diagram that fails to parse falls back to showing
  its raw source (never a broken image).

## Changed
- `DocMarkdown` keeps ` ```mermaid ` fences as raw text (rehype-highlight
  `plainText`) and renders them through a new lazily-loaded `Mermaid` component
  (`frontend/src/docs/Mermaid.tsx`). Mermaid is dynamically imported, so it is
  code-split into its own chunks and only downloaded for docs that actually
  contain a diagram — the main bundle is unaffected. Reuses the existing
  `.docs-rendered .mermaid` styling.
