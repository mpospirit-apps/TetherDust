# 0.4.8 — 2026-06-26

**Deferred fast-follow #8 (Dashboards): scheduled auto-refresh polling.** From
the 0.4.0 API-only + SPA migration's deferred-items list.

## Upgrade notes for admins
- No DB migrations and no new env vars. Rebuild the `frontend` image
  (`docker compose up --build`) to pick up the changes. The
  `auto_refresh` / `refresh_interval` dashboard settings already existed and were
  already saved; this makes the workspace viewer act on them.

## Added
- **Dashboards now auto-refresh on their configured schedule.** When a dashboard
  has **Auto refresh** enabled with a refresh interval (5/15/30 min, 1/6/12/24 h),
  the workspace viewer re-runs each chart on that interval — a real forced refresh
  (`?refresh=1`, the same path as the manual refresh button), not a re-serve of
  the server cache. The dashboard header shows an "Auto-refreshes every …" note
  when polling is active. (Previously the settings were saved but the SPA never
  polled — matching the legacy UI, which also didn't auto-poll.)

## Changed
- `ChartCard` accepts a `refreshIntervalMs` prop and polls via the existing
  refresh mutation; the **initial** paint still uses cached data for speed, and a
  poll is skipped while a refresh for that chart is already in flight.
  `DashboardsPage` derives the interval (minutes → ms) from the dashboard's
  `auto_refresh` + `refresh_interval`. Manual per-chart refresh is unchanged.
