# 0.4.6 — 2026-06-25

**Deferred fast-follow #6 (Chat): `@`-mention + `/`-prompt autocomplete.** From
the 0.4.0 API-only + SPA migration's deferred-items list.

## Upgrade notes for admins
- No DB migrations and no new env vars. Rebuild the `frontend` image
  (`docker compose up --build`) to pick up the changes. The supporting endpoints
  (`/api/v1/chat/doc-sources`, `/api/v1/chat/prompts`) already shipped in 0.4.0;
  this wires them into the chat box.

## Added
- **`@`-mention doc-source autocomplete in the chat box.** Typing `@` (then a
  query) opens a dropdown of the documentation resources you're allowed to see
  (debounced server-side search against the MCP server, role-filtered). Picking
  one drops the trigger text and adds a removable **chip**; the selected
  resources are attached to your message (`resource_uris` / `sources_info`) so the
  agent reads them as context.
- **`/`-prompt slash-command autocomplete.** Typing `/` filters the enabled MCP
  prompts you're allowed to use (loaded once, filtered locally). Picking one adds
  a chip and injects the prompt's content into the message
  (`prompt_context` / `prompts_info`).
- **Selected-mention chips** above the input (remove individually, or backspace on
  an empty box to drop the last one), and **"used" chips** rendered above each
  sent user message — including on reloaded session **history** (the WS already
  persisted `sources_used` / `prompts_used`).
- Keyboard-driven dropdown: ↑/↓ to move, Enter/Tab to pick, Esc to dismiss.

## Changed
- The chat input/composer moved into a dedicated `ChatComposer` component
  (`frontend/src/chat/ChatComposer.tsx`); `useChatSocket.send` now accepts the
  selected resources/prompts and assembles the full WS payload. Ports the legacy
  `mention-chips.js` / `slash-commands.js` UX onto the React chat page.
