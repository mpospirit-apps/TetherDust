# 0.4.14 — 2026-06-26

**Chat UI consistency (SPA polish).** Bring the Chat tab back in line with the
rest of the SPA after the 0.4.0 migration: the history sidebar now uses the same
glass panel as every other tab, and the message stream is restored to the
pre-migration design (avatars, gradient bubbles, tool/mention pills).

## Upgrade notes for admins
- None. Frontend-only change, no DB migrations or env vars. Rebuild the
  `frontend` image (`docker compose up --build`).

## Changed
- **`frontend/src/chat/ChatPage.tsx`** — the chat sidebar now reuses the shared
  `.docs-sidebar` glass panel (header + `.docs-tree` + `.docs-file-btn` rows +
  colored `.history-section-label` group headings) that Docs, Dashboards,
  Reports and Tethers already use, instead of its own minimal `.chat-sidebar`.
  Messages are rendered with the original `.message` / `.message-avatar` /
  `.message-body` / `.message-content` structure so the pre-migration
  `modules/chat.css` styling applies again.
- **`frontend/src/chat/ChatComposer.tsx`** — the composer `<form>` moved off the
  `.chat-input` class (which `modules/chat.css` styles as the legacy textarea
  with `flex: 1`) onto a dedicated `.chat-composer-form`, so it sizes to content
  and centers on the same 900px column as the messages.
- **`frontend/src/styles/app.css`** — dropped the orphaned minimal chat block
  (`.chat-sidebar` / `.chat-msg` / `.chat-bubble` / `.chat-tool` …) and kept only
  the chat-specific glue (agent-status row, session rows, inline status,
  composer row).

## Fixed
- **Sidebar looked nothing like the other tabs** — wrong width, no glass
  background, plain session list. Now identical to the Docs/Dashboards sidebar.
- **Messages lost their avatars, gradient user bubbles and tool/mention pills**
  in the SPA rewrite: the rich `modules/chat.css` rules were orphaned when the
  JSX switched to `.chat-msg` / `.chat-bubble`. Restored to the 0.3.0 design.
- **Composer grew to roughly half the chat height** — the `.chat-input` class
  collision gave the form `flex: 1`. The form no longer inherits that rule.

## Notes
- No `VERSION` bump (stays `0.4.0`), consistent with the other `0.4.x`
  fast-follow notes.
- Verified: `npm run build --prefix frontend` (tsc strict + vite) is clean.
