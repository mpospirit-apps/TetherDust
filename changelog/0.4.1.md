# 0.4.1 — 2026-06-21

**Deferred fast-follow #1 (F2 — Chat): Codex device-login OAuth.** From the 0.4.0
API-only + SPA migration's deferred-items list.

## Upgrade notes for admins
- No DB migrations and no new env vars. Rebuild the `backend` + `frontend` images
  (`docker compose up --build`) to pick up the changes.

## Added
- **Codex device-login (ChatGPT OAuth) in the Agents admin.** A `codex`
  (subscription) agent can now sign in from the browser: the **Authentication**
  panel on the agent edit screen runs the OAuth **device-code** flow — click
  *Sign in to ChatGPT*, open the shown verification URL, enter the code, and the
  SPA polls until you approve. The credential is stored encrypted and refreshed
  automatically; the panel then shows the signed-in account, plan, and token
  expiry, with a *Re-authenticate* button to replace it. Backed by two new staff
  actions, `POST /api/v1/admin/agents/{id}/device-login/` and
  `GET /api/v1/admin/agents/{id}/device-login/{login_id}/`, which proxy the Codex
  gateway's `/auth/device/{start,status}` endpoints. (Previously Codex agents
  could only be configured via a pasted token / API key.)

## Fixed
- **Backend container healthcheck (`/healthz`).** The Finalize URL cleanup in
  0.4.0 dropped the root liveness/readiness probes along with the legacy
  `workspace` views, so the `backend` container's healthcheck 404'd and never
  became healthy — blocking every service that `depends_on` it (`nginx`, celery).
  Restored `/healthz` + `/readyz` as plain views (`api/health.py`, mounted at the
  root in `project/urls.py`).
