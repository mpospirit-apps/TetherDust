# 0.5.3 — 2026-07-05

## Upgrade notes for admins
- Rebuild the frontend and backend after pulling:
  `docker compose up -d --build frontend backend`. No migrations, no new env
  vars.

## Fixed
- The login page logo rendered at its native image resolution (way too
  large) because `.brand-icon` had no size rule. It now reuses the navbar's
  `.nav-logo`/`.nav-title`/`.accent` classes directly, so the logo and
  wordmark match the navbar exactly.
- The login card used a flat, opaque background with a red top accent
  border and a drop shadow, inconsistent with the rest of the app. It now
  uses the same translucent "glass" treatment as the navbar/sidebar (blurred,
  semi-transparent background, hairline border, no shadow) in both light and
  dark mode, and the red border accent was removed.
- The login page had its own reduced background (fewer gradient stops, no
  shooting stars) instead of the app's shared aurora backdrop. It now reuses
  the same `aurora-bg` body class and `AuroraBackground` component (glimmer
  dots + shooting stars) as every other page, and the login card sits above
  them (`z-index: 2`) instead of behind.
- Replaced the sci-fi flavor copy on the login form ("Callsign",
  "Passphrase", "Initiate Access") with plain labels ("Username", "Password",
  "Sign In"), and reworded the subtitle to "Welcome back" so it no longer
  repeats the button text.
- Removed the drop shadows from the admin Mission Control status banner
  (`.mc-health`), KPI cards (`.mc-kpi`), and inventory chips (`.mc-chip`) to
  match the flat glass-card look used elsewhere.
- The Add/Edit Codebase form, and the Register/Generate-page/Generate-library
  documentation forms, were single narrow cards (`maxWidth: 640/760`) leaving
  most of the page empty. All four now use the same two-card `.form-split`
  layout as the Database form, and their page headers drop the "Back" link
  in favor of the primary action + Cancel (wired to the form via the
  `form="..."` attribute, since the buttons now live outside the `<form>`
  element).
- Redesigned the `/admin/docsources/add` chooser (Register / Generate a
  page / Generate a library) from a 2-column grid of opaque, centered square
  cards into a vertical list of full-width rows (icon, title/description,
  chevron), using the same translucent glass treatment as the navbar/sidebar
  instead of a flat `--surface` background.
- The Add/Edit MCP Server form had three cards (`Identity` / `Remote (HTTP)`
  / `Local (subprocess)`) in a 2-column `.form-split`, so the third card
  wrapped onto its own row with empty space beside it. Consolidated Remote +
  Local into one "Connection" card with subsections, matching the
  Identity | Connection pattern used elsewhere; header "Back" replaced with
  Cancel + Create Server/Save Changes.
- Applied the same fixes (two-card `.form-split` layout, header Cancel +
  primary action instead of "Back") to Add/Edit Agent, Add/Edit Database
  Connection, New/Edit Report, New/Edit Dashboard, and Add/Edit Chart, plus
  the AI-generation forms (Generate Dashboard, Generate Documentation page,
  Generate Documentation library) — all previously single narrow cards
  (`maxWidth`) leaving most of the page empty, or leaving a "Back"-only
  header with no primary action promoted.
- Polished the `/admin/agents/new` integration-type picker to match the docs
  chooser: replaced opaque cards of wrapped chip buttons with the same
  translucent glass list-row style (icon, label, chevron), grouped under
  category headings with a one-line hint per category. Generalized the
  underlying CSS from docs-specific `.doc-choice-*` classes into shared
  `.choice-list`/`.choice-card`/`.choice-section` classes in `layouts.css`.
- Applied the same two-card `.form-split` layout and header Cancel + primary
  action (instead of "Back") to the New/Edit Tether form ("Identity" +
  "Sources & access" cards).
- The Settings page had two separate "Save General" / "Save SMTP" buttons for
  what admins experience as one settings page. Merged both sections into a
  single form with one "Save Settings" button in the page header, saving
  both in parallel; each card still shows its own error if only one save
  fails. "Send Test Email" remains a separate action, since it isn't a save.
- Applied the same two-card `.form-split` layout and header Cancel + primary
  action (instead of "Back") to the Add/Edit Role form (already split, just
  needed the header fix) and the Add/Edit User form (split into "Account"
  and "Access" cards; was a single narrow `maxWidth: 560` card).
- Mission Control: reduced the vertical gap between the status banner, KPI
  row, inventory chips, and trends row (`margin-bottom` `var(--lg)` →
  `var(--md)`); removed the status banner's colored left border and its
  per-status color overrides; removed the query-volume bar graph's
  hover-to-pink color swap.
- Mission Control's "Recent Queries" table dropped the "Rows" and "ms"
  columns (too detailed for an overview page), and "Recent Chat Sessions"
  dropped "Msgs" and renamed "Last Activity" to "When", matching the
  Recent Queries table's column naming.
- Mission Control's "Recent Chat Sessions" showed newly created, empty chat
  sessions (0 messages) as "Untitled", since a session's title is only set
  from its first message. The `/admin/sessions` page already filtered these
  out; applied the same `message_count > 0` filter to the Mission Control
  overview endpoint (`api/v1/admin/overview.py`).
