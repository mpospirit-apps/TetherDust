# 0.5.5 — 2026-07-07

## Upgrade notes for admins
- Rebuild the frontend after pulling: `docker compose up -d --build frontend`.
  No migrations, no new env vars.

## Added
- Collapsible "How it works" explainers on every content-creation form,
  reusing one `.doc-hiw-card` component (click anywhere on the card to
  expand/collapse, animated via a CSS grid-rows transition):
  - `/admin/docsources/generate` and `/admin/docsources/library` restore the
    step-by-step explainer that existed pre-SPA (in the old server-rendered
    console), with the original copy for each.
  - `/admin/docsources/register`, `/admin/reports/new`, `/admin/tethers/new`,
    `/admin/dashboards/generate`, `/admin/dashboards/new`, and "Add Chart"
    get new explainers (none of these flows existed pre-SPA), written from
    and verified against the actual mechanics in `report_engine.py`,
    `tether_engine.py`, and the `save_tether_graph`/`add_chart`/
    `create_dashboard` MCP tools.
  - Shown only in create mode; hidden when editing an existing item.
- `/admin/dashboards/add`: a new in-between choice page (mirroring
  `/admin/docsources/add`) with "New dashboard" and "Generate with AI" cards.
  The dashboards list's "Generate with AI" / "+ New Dashboard" buttons are
  now a single "+ Add Dashboard" button that opens this picker; canceling out
  of "New Dashboard" returns here instead of the main list.

## Changed
- "+ New Report" and "+ New Tether" buttons renamed to "+ Add Report" and
  "+ Add Tether" for consistency with "+ Add Dashboard" / "+ Add
  Documentation" / "+ Add Agent".
- Agents list: the "Activate" action for an inactive agent now lives in the
  Active column (replacing the muted `—` placeholder) instead of the Actions
  column, so Active reads as a single-glance status-or-action cell.

## Fixed
- Report Runs page had a leftover "Reports" ghost button in the header that
  just duplicated the top-nav link; removed.
- `<button class="btn">` elements (e.g. "Save Settings") rendered ~5px
  shorter than `<a class="btn">` elements (e.g. "+ Add Dashboard") despite
  identical padding — browsers apply a UA-stylesheet `line-height: normal`
  to form controls that isn't overridden anywhere, while anchors just
  inherit the body's `line-height: 1.6`. Added `line-height: 1.6` to the
  shared `.btn` rule so every button- and link-based CTA in the app is the
  same height.
