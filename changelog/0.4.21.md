# 0.4.21 — 2026-06-30

## Upgrade notes for admins
- Rebuild images after pulling: `docker compose up -d --build`. The per-service
  Docker build directory was renamed `docker/` → `containers/`; Compose, the
  Dockerfiles, and CI already point at the new path, so a standard rebuild picks
  it up. No migrations, no new env vars. If you maintain a local Compose
  override or custom tooling that references `docker/<service>/…` paths, update
  those references to `containers/<service>/…`.

## Changed
- Renamed the `docker/` directory to `containers/` to better describe its
  contents (per-service container build contexts and gateway/MCP wrapper apps:
  `codex`, `claude`, `local_mcp`, `nginx`, `tdmcp`). The old name collided
  conceptually with the `docker compose` tooling and the root `Dockerfile`.
  Updated all references: `docker-compose.yml` build contexts/dockerfiles, the
  root `Dockerfile` `COPY` lines, each service Dockerfile's internal `COPY`
  paths, the runtime AGENTS.md/CLAUDE.md lookup in
  `backend/api/v1/admin/agents.py`, lint/type-check targets (`ci.yml`,
  `.pre-commit-config.yaml`, `pyrefly.toml`), Dependabot Docker directories, and
  the docs/changelog references.
