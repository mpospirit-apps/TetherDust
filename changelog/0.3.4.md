# 0.3.4 — 2026-06-20

Second feature vertical of the **API-only backend + React/Vite SPA** migration:
**Dashboards** (V2) — the workspace dashboard viewer with live D3 chart rendering,
plus admin dashboard/chart CRUD, the SQL/D3 chart editor with AI assist, and AI
dashboard generation. Runs **alongside** the legacy server-rendered UI.

## Upgrade notes for admins
- No DB migrations, no new env vars.
- Rebuild the `frontend` image so the SPA picks up the new `d3` (+ `@types/d3`)
  dependency — d3 is now **bundled** with the SPA instead of loaded from a CDN, so
  charts render with no external network dependency (`docker compose build frontend`).

## Added
- **Dashboards viewer (workspace SPA)** at `/dashboards` — a searchable dashboard
  sidebar and a 12-column chart grid that executes each chart's stored `custom_d3_code`
  client-side (the existing `new Function(data, container, d3, theme)` rendering,
  preserved), with per-chart cached-data + manual refresh. Backed by role-scoped
  `GET /api/v1/dashboards/`, `/dashboards/<id>/`, and `/charts/<id>/data/`.
- **Dashboards admin (SPA)** under `/admin/dashboards`, backed by
  `/api/v1/admin/{dashboards,charts}`: dashboard CRUD (name, description, auto-refresh,
  allowed roles) and chart CRUD. The **chart editor** has a SQL editor with an ad-hoc
  **Run Query** preview (`charts/preview/`, read-only-validated), a live D3 preview that
  re-renders as you edit, and — when editing an existing chart — an **AI assist** panel
  over the `ws/chart-edit/<id>/` WebSocket that edits the chart in place via the
  `update_chart` tool and reloads the saved state.
- **AI dashboard generation** at `/admin/dashboards/generate` (pick databases / docs /
  codebases + a style + an agent; live status polling), backed by the `generate` /
  `generate-options` actions, with run history under `/api/v1/admin/chartgen-logs`
  (the **Chart Generation** monitor at `/admin/chartgen-logs`).

## Changed
- Chart data execution + ad-hoc preview (read-only SQL validation, query run, result
  caching) moved out of the legacy views into reusable `engine/engines/charts.py`; AI
  dashboard-generation orchestration moved into `engine/engines/chartgen.py` (unchanged
  behaviour — runs on the **active** agent in a background thread).

## Fixed
- **AI-generated charts rendered blank**: the agent emits `custom_d3_code` as a bare function
  *expression* (`(function(data, container, d3, theme){…})`) rather than a function *body*. Run as
  the body of `new Function(...)` it is defined and discarded — never called — so nothing rendered
  and no error showed (a latent issue inherited from the legacy renderer). The renderer
  (`src/charts/render.ts`, shared by the dashboard cards and the editor preview) now falls back to
  evaluating such code to its function and invoking it; function-body and self-invoking-IIFE charts
  are unaffected.
- **The AI chart editor was unreachable**: the `ws/chart-edit/` WebSocket route matched
  only integer IDs (`(?P<chart_id>\d+)`) and the consumer did `int(chart_id)`, but chart
  IDs are prefixed strings (`cht_…`) — so the route never matched and the consumer would
  have rejected the connection. Both now accept the string IDs (the same class of
  leftover-integer-ID bug fixed for the local-MCP proxy in 0.3.2).
