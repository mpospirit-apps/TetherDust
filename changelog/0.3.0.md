# 0.3.0 — 2026-06-17

## Upgrade notes for admins
- **Breaking — fresh database required.** Model primary keys changed from
  integers to prefixed string IDs (`agt_`, `db_`, …) and the `0001_initial`
  migration was reset. Existing data is not migratable in place; start from a
  clean database (drop/recreate the app DB volume, then run migrations).
- **`manage.py` path changed.** The Django app moved to `backend/`. Run
  migrations with
  `docker compose run --rm web python backend/manage.py migrate`.
- **Repository layout flattened.** `pyproject.toml`, `uv.lock`, `.venv`,
  `tests/`, and the MCP server now live at the repository root. If you have
  local checkouts, scripts, or CI referencing the old `tetherdust/`,
  `mcp_server/`, or `web/` paths, update them.
- No new required environment variables.

## Changed
- Renamed Django apps for clarity: `core` → `engine`, `portal` → `workspace`,
  `console` → `management`.
- Flattened the repository to the root: moved the web app to `backend/`
  (`tetherdust_web/` → `project/`), relocated `tests/`, package metadata
  (`pyproject.toml`, `uv.lock`, `.venv`), and `mcp_server/` (now `tdmcp/`) to
  the root, and aligned service, container, and configuration naming. Updated
  Dockerfiles, the CI workflow, hatchling/pytest/coverage paths, and
  documentation references accordingly.
- Reset all 21 engine models to the dj-models convention (Meta-first,
  `verbose_name(_plural)`, field reordering, constraints/indexes in Meta) and
  switched to prefixed string primary keys via `engine/ids.py`; URL converters
  and view signatures updated from `<int:>` to `<str:>`.
- Replaced `_x` + property encrypted fields with dedicated encrypted model
  fields (`engine/models/fields.py`).
- Extracted all business logic into `engine/services/` behind a `get()`
  registry; repointed ~140 call sites, context processors, and templates.
- Cleaned up imports and loggers across `engine`, `management`, `workspace`,
  `tdmcp`, and `docker` for PEP8 compliance.
- Added ruff, mypy, and pyrefly configuration; excluded agent skill files via
  `.gitignore`.

## Fixed
- Resolved lint and type-check issues across the MCP server, web app, and
  tests (type annotations and import statements).
- Updated `tdmcp` raw-SQL inserts/casts, the docker entrypoint, and credential
  forms for the new string-PK and encrypted-field interface.
