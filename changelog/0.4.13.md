# 0.4.13 — 2026-06-26

**Backend fast-follow #4 (move logic off the `Tether` model).** Relocate the
computed properties and validation from the `Tether` model to the service/API
edge, leaving the model as data + `__str__` only.

## Upgrade notes for admins
- None. No DB migrations (only methods/properties were removed — no field
  changes), no new env vars. Rebuild the `backend` image (`docker compose up
  --build`).

## Added
- **`engine/services/tether.py` — `TetherService.source_name(tether)`.** The
  former `Tether.source_name` property, now a service method (registered through
  the existing `get()` registry and exported from `engine.services`).

## Changed
- **`engine/models/tethers.py`** — removed `Tether.source_name`,
  `Tether.uses_codebase_repo`, and `Tether.clean()` (and the now-unused
  `ValidationError` import). The model is back to fields + `__str__`.
- **`api/v1/admin/tethers.py`** — `source_name` is now a `SerializerMethodField`
  backed by `TetherService` (mirrors the existing `latest_version` field).
- **`api/v1/tethers.py`** — the public tether summary now computes `source_name`
  via `get(TetherService)` instead of the model property.

## Removed
- **`Tether.uses_codebase_repo`** — had zero consumers anywhere in the codebase.

## Notes
- **No behavior change.** `Tether.clean()` (the "exactly one code source" rule)
  was already dead code — nothing called `full_clean()`, and the admin
  `TetherSerializer.validate()` already enforces the identical rule on the write
  path. The public API contract is unchanged: `source_name` still renders in both
  the public (`/api/v1/...`) and admin (`/api/v1/admin/tethers/`) responses.
- Verified: ruff/mypy/pyrefly clean, `manage.py makemigrations --check` reports
  no changes, and a smoke test confirms the serializer builds and
  `TetherService.source_name` returns the right value.
