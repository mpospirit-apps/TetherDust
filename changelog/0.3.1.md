# 0.3.1 — 2026-06-20

In-progress work toward an **API-only backend + React/Vite single-page app**. The new
API and SPA run **alongside** the existing server-rendered UI — nothing has been removed
yet, so the legacy app keeps working. (Migration foundations + the admin console + agent/chat;
the MCP-server work and the user-facing feature pages land in later notes.)

## Upgrade notes for admins
- No DB migrations and no new required env vars.
- New container topology (dev): `backend` (Django/Daphne, renamed from `web`), `frontend`
  (compiled React, served by its own nginx), and `nginx` (public gateway). The **new SPA** is
  served at **http://localhost:8080**; the **legacy UI** stays at **http://localhost:8000**.
- New Python deps: `djangorestframework` (in the `web` extra) and `djangorestframework-stubs`
  (in the `dev` extra). Rebuild images: `docker compose build`.

## Added
- Django REST Framework and a consolidated `backend/api/` app exposing the new JSON API under
  `/api/` (session-cookie auth + CSRF; no CORS — single origin).
- Auth API `/api/v1/auth/{csrf,login,logout,me}` — `me` returns the per-user capability flags
  the SPA uses for route guards (replacing the template context processors).
- DRF permission classes wrapping `engine.services.PermissionService` (`CanChat`,
  `CanViewDocs/Reports/Dashboards/Tethers`, `IsStaffUser`, `CanManageUsers`).
- New `frontend/` React 19 + Vite + TypeScript app with the existing design system ported
  verbatim (theme tokens, light/dark, aurora background, navbar/dropdown, restyled login) and
  permission-gated routing.
- **Admin console (SPA)** under staff-gated `/admin`, backed by `/api/v1/admin/*`:
  Databases (CRUD + connectivity test), Settings (general + SMTP + test email), Roles (CRUD +
  access-grant multi-selects, admin-role → staff sync), Users (CRUD + role assignment), **Agents**
  (CRUD + activate + per-type auth: API key / pasted Claude OAuth token / direct base-URL+model),
  and read-only Audit Log + Chat Sessions viewers.
- **Chat workspace (SPA)** — real-time agent chat over the existing `ws/chat/` WebSocket: streaming
  responses, tool-call status, a session sidebar (new / switch / delete), markdown rendering, and an
  agent-status indicator. Backed by `/api/v1/agent-status/` and
  `/api/v1/chat/{sessions,doc-sources,prompts}`.
- Frontend deps `react-markdown` + `remark-gfm` for rendering agent responses.
- `nginx` gateway container.

## Changed
- Split the stack into three containers: renamed `web` → `backend`, added a `frontend`
  container that builds and serves the SPA, and an `nginx` gateway (`:8080`) that routes
  `/api` + `/ws` → `backend` and everything else → `frontend`. `frontend/` owns its own
  Dockerfile + nginx config; `docker/nginx/` is the gateway.

## Fixed
- nginx gateway now forwards the full `Host` header via `$http_host` (which keeps the `:8080`
  port) instead of `$host` (which strips it), so Django's CSRF Origin check matches the
  browser origin — authenticated writes from the SPA were previously rejected with
  "Origin checking failed".
- The SPA now surfaces DRF field-validation errors (e.g. an invalid or duplicate username)
  in admin forms instead of a generic "Save failed".
