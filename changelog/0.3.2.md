# 0.3.2 — 2026-06-20

Continues the **API-only backend + React/Vite SPA** migration: the MCP-server admin
section and the internal service API that decouples the MCP server's writes from the
Django schema. Like 0.3.1, this runs **alongside** the legacy server-rendered UI — nothing
has been removed yet.

## Upgrade notes for admins
- No DB migrations.
- **New required env var: `INTERNAL_API_SERVICE_TOKEN`** — a shared secret the MCP server uses
  to authenticate its writes to the backend's internal API. Generate one
  (`python -c "import secrets; print(secrets.token_urlsafe(32))"`) and add it to `.env`. Without
  it the backend rejects every `/api/internal/` call, so the agent's dashboard/chart/tether
  write tools will fail.
- `httpx` moved into the base dependencies (the MCP server now needs it on the write path), and
  the `tdmcp` + `backend` + `celery` services gained `INTERNAL_API_SERVICE_TOKEN` /
  `BACKEND_INTERNAL_API_URL`. Rebuild images: `docker compose build`.
- `backend` is appended to the in-container `DJANGO_ALLOWED_HOSTS` so the MCP server's call to
  `http://backend:8000/api/internal` passes Django's Host-header check. The public origin is
  unaffected (the gateway blocks `/api/internal/` — it is in-network only).

## Added
- **MCP servers admin (SPA)** under `/admin/mcp-servers`, backed by
  `/api/v1/admin/{mcp-servers,mcp-prompts}`: register/edit custom MCP servers (remote HTTP or
  local subprocess, encrypted bearer token + command env), a connectivity test (initialize +
  tools/list probe that now reports the probed URL), a read-only tool list, and prompt-template
  CRUD + enable/disable toggle. Saving/deleting a local server notifies the `local-mcp` container
  to reload.
- **Internal service API** at `/api/internal/` (new `ServiceTokenAuthentication`, `X-Service-Token`
  header; fails closed when the token is unset): `POST /dashboards`, `POST /dashboards/{id}/charts`,
  `PATCH /charts/{id}`, and `POST /tether-versions/{id}/graph`. Reuses the existing read-only-SQL
  and tether-graph schema validation from `engine`.

## Changed
- The MCP server's mutating tools (`create_dashboard`, `add_chart`, `update_chart`,
  `save_tether_graph`) now persist through the backend's internal API over HTTP instead of
  writing to Postgres directly via raw SQL — `tdmcp` no longer has any schema coupling to the
  Django models on the write path. (Its read-side config loading still queries the DB directly;
  that's unchanged.)
- The public nginx gateway returns 404 for `/api/internal/`, keeping the service-token write
  endpoints off the public origin; the MCP server reaches them in-network only.

## Fixed
- **Local (subprocess) MCP servers were unusable**: the `local-mcp` proxy typed its route and
  registry as integer IDs (`proxy_mcp(server_id: int)`, `_proxies: dict[int, …]`), left over from
  when model IDs were integers. Server IDs are now prefixed strings (`mcp_…`), so FastAPI rejected
  every request with HTTP 422 before the handler ran — breaking both the admin "test" and the
  agent's use of local servers. Corrected the annotations + path converter to `str` (the runtime
  already stored string IDs throughout).
- The MCP-server connectivity test now shows **which URL it probed** in the result, so an
  unreachable remote (e.g. a `localhost`-based URL that resolves to the backend container inside
  Docker, not your host) is self-diagnosing instead of a bare "Connection refused".
