# 0.4.0 — 2026-06-21

**API-only backend + React/Vite SPA — cutover.** This release completes the
migration: the Django backend is now **API-only** (Django REST Framework under
`/api/` + the unchanged Channels WebSocket layer), the entire UI is the **React
SPA** served through the **nginx** single-origin gateway, and the `tdmcp` mutating
tools persist through the internal service API. The legacy server-rendered
`workspace`/`management` pages, the `/mission-control/` prefix, the Django HTML
forms, the UI context processors, and the old static JS are **removed**.

The feature work that landed during the migration is summarised in
`changelog/0.3.1`–`0.3.6` (admin core, agent + chat, MCP internal API, docs,
dashboards, reports, tethers); this entry covers the cutover itself.

## Upgrade notes for admins
- **The app now lives on `http://<host>:8000`** — this is the **nginx gateway**
  serving the SPA and proxying `/api` + `/ws/` to the backend. The backend
  (Daphne) is **internal-only** and no longer publishes a host port. There is no
  longer a separate `:8080`; the old server-rendered UI is gone.
- **Rebuild and restart the whole stack**: `docker compose up --build`. The
  `frontend` + `nginx` images must be rebuilt; the `backend` image no longer
  serves any HTML.
- If you set `DJANGO_CSRF_TRUSTED_ORIGINS` / a reverse proxy in front, point it at
  the **nginx** service on `:8000` (not the backend). TLS terminators should keep
  forwarding the full `Host` header (the gateway already does via `$http_host`).
- No DB migrations and no new env vars. Existing data is unaffected.

## Removed
- Legacy server-rendered UI: `workspace/` and `management/` `views/`, `templates/`,
  `urls.py`, and `context_processors.py`; `management/forms/` (Django HTML forms);
  `workspace/templatetags/`; and `backend/static/js/` (the old vanilla-JS SPA). The
  `mission-control/` and `""` URL includes and the dev `staticfiles_urlpatterns()`
  are dropped — only `path("api/", …)` remains. Each app keeps its `routing.py` +
  `consumers/` (the WebSocket layer is unchanged).
- The seven UI template context processors were removed from `TEMPLATES`; the
  permission flags they injected are now served by `GET /api/v1/auth/me/`. The
  Django template engine itself is retained for DRF's browsable API.

## Changed
- **nginx is the single public origin on `:8000`** (was `:8080` during the
  migration); `backend` moves to `expose`-only. The gateway still forwards
  `$http_host` so Django's CSRF Origin check matches the browser origin.
- `VERSION` → `0.4.0`; `README` + `CLAUDE.md` updated to describe the API-only
  backend + SPA architecture (no more `web/` server-rendered app).
