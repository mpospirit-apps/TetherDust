# 0.4.5 — 2026-06-25

**Deferred fast-follow #5 (Agents): system-prompt auto-seed.** From the 0.4.0
API-only + SPA migration's deferred-items list.

## Upgrade notes for admins
- No DB migrations and no new env vars. Rebuild the `backend` + `frontend` images
  (`docker compose up --build`) to pick up the changes. The default-prompt files
  (`docker/codex/AGENTS.md`, `docker/claude/CLAUDE.md`) are already copied into the
  backend image, so no extra setup is needed.

## Added
- **System Prompt auto-seed (Console → Agents).** Opening the agent form for a new
  agent — or editing an existing agent whose **System Prompt is blank** — now
  pre-fills the textarea from the baked-in container default
  (`docker/claude/CLAUDE.md` for Claude Code agents, `docker/codex/AGENTS.md` for
  every other CLI / Direct-API type), so you see and can edit the prompt the agent
  effectively uses instead of an empty box. The pre-fill is **display only** — it
  is persisted only if you save the form, and it never overwrites a prompt you've
  already typed or an agent's existing non-blank prompt (it seeds at most once per
  type). Backed by a new `GET /api/v1/admin/agents/default-prompt/?agent_type=…`
  action. (Restores the legacy agent-form pre-fill, which wasn't ported when the
  Agents admin shipped in 0.4.0.)

## Changed
- The System Prompt field help text now explains that the value is pre-filled from
  the container default and that clearing it falls back to the container default.
