# 0.5.2 — 2026-07-04

## Upgrade notes for admins
- Rebuild the frontend after pulling: `docker compose up -d --build frontend`.
  No migrations, no new env vars.

## Changed
- Restyled Mermaid diagrams and fenced code blocks in the docs viewer to use
  the TetherDust accent palette instead of Mermaid/highlight.js defaults:
  cyan participant boxes, orange notes, pink labels/activations for diagrams;
  pink keywords, lime strings, orange numbers, cyan built-ins for code (with
  a `color-mix`-darkened variant of each accent in light mode, since the raw
  accents — especially lime — don't have enough contrast against a pale
  background).
- Code blocks now share the same translucent "glass" card treatment as the
  navbar/sidebar (in both light and dark mode) instead of a flat background,
  gained a language label + copy-to-clipboard button in the top-right
  corner, dropped a leftover drop shadow, and bumped their font size by 1px.
- Extended the same code-block styling (and syntax highlighting) to chat
  messages, which previously had no syntax highlighting at all and used a
  separate hardcoded dark/cyan-accent look.
- Chat composer, and both agent and user message bubbles, now use the same
  translucent "glass" treatment (blurred, semi-transparent background plus
  hairline border, in both light and dark mode) as the navbar and sidebar,
  instead of a flat background — user bubbles previously used a solid
  red/pink gradient, which is now the same glass card as everything else.

## Fixed
- Mermaid diagrams could sometimes render incorrectly or fall back to raw
  source text on docs with multiple diagrams: `mermaid.render()` isn't safe
  to call concurrently, and each diagram is its own component mounting at
  once. Renders are now serialized through a shared queue.
- `useTheme()` gave every component its own isolated state, so toggling
  dark/light in the navbar never notified other consumers (Mermaid,
  dashboard charts) to re-render — a diagram rendered once stayed stuck with
  whatever theme was active at mount. It's now a shared external store via
  `useSyncExternalStore`, so all consumers react to the same change.
- Code blocks in chat messages overflowed past the message bubble (and
  off-screen) with long lines instead of scrolling within the bubble, similar
  to the table overflow fixed in 0.5.1. Root cause was two-fold: the block
  wrapping the code needed an explicit width to stop a flex-sized bubble
  from stretching to fit a long line, and the element carrying the
  card's background/border was sized to the visible viewport rather than
  the actual (scrollable) content width, so scrolling revealed bare text
  past the card's own border. Fixed both, and used a viewport-relative
  (non-circular) width cap so short code blocks still fill the bubble's
  full width — including bubbles whose only content is a code block —
  instead of shrinking to the block's own content width.
- The first message in a chat sat flush against the navbar with no gap
  (the messages list's top padding was exactly the navbar's height). Added
  breathing room above it, matching the same `navbar-h + spacing` pattern
  used elsewhere in the app.
- Removed the chat composer's inner padding (replaced with bottom margin)
  so its glass card hugs the textarea. A leftover duplicate
  `.chat-composer-form` rule in `app.css` — imported after the module that
  actually owns this class — was silently re-adding the old padding via
  cascade order; removed the dead rule.
