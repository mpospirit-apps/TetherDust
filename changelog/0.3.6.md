# 0.3.6 — 2026-06-21

Fourth (and final) feature vertical of the **API-only backend + React/Vite SPA**
migration: **Tethers** (V4) — the workspace code ↔ database graph canvas plus
admin **codebases** (add / sync) and **tethers** (CRUD + agent-driven generation
with live status). The d3-force two-lane canvas (tapered-bezier edges, pan / zoom
/ search / relationship-filter / spread+gap controls) is ported from the legacy
vanilla-JS modules into a React-driven controller. Runs **alongside** the legacy
server-rendered UI; with this vertical every workspace + admin surface now exists
in the SPA, clearing the way for the legacy-removal cutover.

## Upgrade notes for admins
- No DB migrations, no new env vars.
- Rebuild the `frontend` image so the SPA picks up the new tethers pages
  (`docker compose build frontend`); no backend dependency changes.

## Added
- **Tethers viewer (workspace SPA)** at `/tethers` — a searchable tether sidebar
  and the interactive **ER-style canvas**: code-file / db-table cards in two lanes
  driven by a d3-force simulation, animated tapered-bezier edges coloured by
  relationship, row-level edge anchoring, hover tooltips, a selection details
  panel (1-hop focus), search, per-relationship filters, a graph-info panel
  (code/db summaries), spread + gap layout sliders (persisted), pan / zoom / fit,
  and card drag-to-pin. Backed by role-scoped `GET /api/v1/tethers/`,
  `/tethers/<id>/`, and `/tethers/<id>/graph/` (the stored current-version graph).
- **Codebases admin (SPA)** under `/admin/codebases`, backed by
  `/api/v1/admin/codebases`: GitHub repository CRUD (URL validated to owner/repo,
  branch / subpath / include+exclude globs, encrypted access token) with an
  on-demand **Sync** that caches the repo file tree via the Celery `sync_codebase`
  task; the list polls sync status while a sync is in flight.
- **Tethers admin (SPA)** under `/admin/tethers`, backed by `/api/v1/admin/tethers`:
  tether CRUD (code side = a live codebase **or** a codebase doc source, validated
  exactly-one; database side = a database doc source; allowed roles), and a detail
  page with the **versions** history, a **Regenerate** action, and **live agent
  output** polled from the `status` action while a generation run is in flight.
  Creating a tether (and Regenerate) starts a background run on the active agent.

## Changed
- Tether generation orchestration (create a `TetherVersion` + run the generator in
  a background thread) moved out of the legacy `management/views/tether.py` into a
  reusable `engine.engines.tether_engine.start_generation` (unchanged behaviour) —
  shared by the create and regenerate endpoints.
- The d3-force canvas is now **bundled** with the SPA (d3 from npm, ported into
  `src/tethers/`) instead of loading `d3-force` from a CDN at runtime; the imperative
  logic is a faithful port restructured as a controller a React component mounts and
  tears down (listeners + RAF loop + simulation cleaned up on unmount).
