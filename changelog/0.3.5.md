# 0.3.5 — 2026-06-21

Third feature vertical of the **API-only backend + React/Vite SPA** migration:
**Reports** (V3) — the workspace report viewer (latest/history results, CSV/Excel
download, email-to-self) plus the admin report-definition CRUD with scheduling,
manual run / preview, and an execution monitor. No agent/MCP in this vertical;
scheduled runs and delivery keep using the existing Celery tasks + SMTP from F1.
Runs **alongside** the legacy server-rendered UI.

## Upgrade notes for admins
- No DB migrations, no new env vars.
- Rebuild the `frontend` image so the SPA picks up the new reports pages
  (`docker compose build frontend`); no backend dependency changes.

## Added
- **Reports viewer (workspace SPA)** at `/reports` — a report sidebar grouped by
  last-run time (Today / Yesterday / weekday / month) plus a results pane showing the
  latest successful run (status, row count, duration), a **History** view of recent
  executions, per-execution **CSV / Excel download** (same-origin `<a href>`, cookie
  auth), and **email-to-self** (when SMTP is configured). Backed by role-scoped
  `GET /api/v1/reports/`, `/reports/<id>/{latest,history}/`, `/executions/<id>/`,
  `/executions/<id>/download/<fmt>/`, and `POST /executions/<id>/send-email/`.
- **Reports admin (SPA)** under `/admin/reports`, backed by `/api/v1/admin/reports`:
  report-definition CRUD (name, description, database, read-only SQL, schedule, delivery,
  allowed roles), an **active/inactive** toggle, a manual **Run**, and a **Preview
  (10 rows)** that runs the saved SQL without persisting a scheduled run. Scheduling
  supports manual / interval / daily / weekly / monthly with `next_run_at` recomputed on
  every save (the existing Celery beat `check_due_reports` picks them up unchanged).
- **Report Runs monitor** at `/admin/report-runs` (backed by
  `/api/v1/admin/report-executions`) — a polling list of recent executions with status,
  duration, row count, and who triggered them, linking to a per-run results view.

## Changed
- Report result rows are now served as a **capped preview** (25 rows for viewing, 10 for
  the admin preview) with `row_count` carrying the true total; full data is streamed only
  on download. The public reports endpoints reuse `engine/engines/{report_engine,
  result_storage,email_service}` and the role scoping in `PermissionService` —
  `workspace/views/reports.py` + `management/views/report.py` are ported to DRF with no
  behaviour change.
