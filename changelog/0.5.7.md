# 0.5.7 — 2026-07-07

## Upgrade notes for admins
- Run the migration after pulling: `docker compose run --rm backend python backend/manage.py migrate`.
  No new env vars.

## Added
- **GitLab codebases**: `Codebase` now supports GitLab (gitlab.com only, no
  self-managed instances) alongside GitHub — same on-demand, no-clone model
  (live REST reads, Celery-cached file tree).
  - **Add Codebase** (`/admin/codebases/new`) now opens with a provider
    picker (GitHub / GitLab), matching the database engine picker on
    **Add Database Connection**. Existing codebases keep working unchanged
    (default provider is GitHub); edit forms gained a **Provider** selector.
  - The codebase MCP tools (`read_codebase_file`, `search_codebase`,
    `list_codebases`) transparently dispatch to the right provider's API.

## Changed
- Extracted the provider-agnostic glob-filtering helpers (`filter_tree`,
  `matches_any`) out of `github_client.py` into a shared
  `engine/integrations/tree_filter.py`, reused by both the GitHub and GitLab
  clients.
