# 0.3.3 — 2026-06-20

Continues the **API-only backend + React/Vite SPA** migration with the first feature
vertical: **Documentation** (V1) — the workspace docs viewer and the admin doc-source
management + AI generation. Like 0.3.1/0.3.2 this runs **alongside** the legacy
server-rendered UI; nothing has been removed yet.

## Upgrade notes for admins
- No DB migrations, no new env vars.
- Rebuild the `frontend` image so the SPA picks up the new `rehype-highlight` dependency
  (`docker compose build frontend`). It is added to `frontend/package.json` +
  `package-lock.json`; `npm ci` in the image build installs it.

## Added
- **Docs viewer (workspace SPA)** at `/docs` — a role-scoped source tree plus a raw-Markdown
  renderer. Backed by `GET /api/v1/docs/sources/` (sources + nested file trees, role-filtered)
  and `GET /api/v1/docs/content/` (raw Markdown or raw source for code files + metadata). The
  API returns **raw Markdown**; React renders it with `react-markdown` + `remark-gfm` +
  `rehype-highlight`, and the custom `[[Source/path|Display]]` **WikiLink** syntax is ported to a
  remark plugin that rewrites links to `/docs/<source>/<path>` SPA routes (unknown/unauthorised
  targets render inert, matching the old server behaviour). Code files render syntax-highlighted.
- **Documentation admin (SPA)** under `/admin/docsources`, backed by
  `/api/v1/admin/docsources`: register an existing folder as a typed source, edit
  type/description/file-patterns, delete (also removes the folder on disk), and a `validate`
  action reporting matched-file count + last-modified. An add picker routes to the three flows.
- **AI documentation generation (SPA)** — single-page (`/admin/docsources/generate`) and
  multi-page **library** (`/admin/docsources/library`) flows: pick source databases / docs /
  codebases + an agent, start generation, and poll live status to a result/preview panel. Backed
  by the `generate`, `generate-library`, and `generate-options` actions, with run history +
  status polling under `/api/v1/admin/docgen-logs` (surfaced as the **Doc Generation** monitor at
  `/admin/docgen-logs`).

## Changed
- The documentation-generation orchestration (single-file + library: prompt assembly, the
  background agent run that streams status into `DocGenerationLog`, and the result/library
  folder scan) moved out of the legacy `management/views/docsource.py` into a reusable
  `engine/engines/docgen.py`, so the API drives it without depending on the to-be-deleted view
  module. Behaviour is unchanged — generation still runs on the **active** agent in a background
  thread, with the selected agent recorded on the log for audit.
