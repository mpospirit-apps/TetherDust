# 0.4.9 — 2026-06-26

**Deferred fast-follow #9 (Finalize/static): remove the dead legacy static
assets.** From the 0.4.0 API-only + SPA migration's deferred-items list.

## Upgrade notes for admins
- No DB migrations, no new env vars, no settings or build-pipeline changes.
  Rebuild the `backend` image (`docker compose up --build`) to drop the removed
  assets from the collected static set.

## Removed
- **Deleted the dead legacy static tree under `backend/static/`** — left over from
  the removed server-rendered UI: `css/` (the old design system, ~16 files),
  `images/` (the logo + DB/agent **brand-icons**), and the legacy FontAwesome
  bundle (`fontawesome/all.min.css` + `webfonts/`). 0.4.0 only removed
  `backend/static/js/`; these remained and were collected into the image's static
  set for nothing.

## Audit (why this is safe)
- Nothing in the backend references them: no template, view, or setting points at
  `tetherdust.css`, the brand-icons, or the FontAwesome bundle (only the generic
  `STATIC_URL` / `STATICFILES_DIRS` remain).
- The SPA has its **own** copies under `frontend/public/images/` (logo +
  brand-icons) and references them as `/images/...`, served by the **frontend**
  container — never the backend. The nginx gateway routes only `/api` + `/ws/` to
  the backend; everything else (incl. `/static/`) goes to the frontend.
- The Django admin is not installed; the DRF **browsable API** static comes from
  the `rest_framework` package (via `AppDirectoriesFinder`), not `backend/static/`.
  `collectstatic` still gathers those 27 files; `manage.py check` is clean.
- `backend/static/` is kept as an empty directory (a `.gitkeep`) so the existing
  `STATICFILES_DIRS` entry and the Dockerfile's `mkdir -p backend/static` stay
  valid with no warning — no settings or Dockerfile change was needed.
